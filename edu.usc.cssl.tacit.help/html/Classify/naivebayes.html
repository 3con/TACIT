Naive Bayes Classifier
Overview
Naive Bayes methods are a family of supervised learning algorithms that are used to classify data instances according to a finite set of class types. Naive Bayes classification is accomplished by applying Bayes theorem with the strong assumption that dependent features (i.e. dependent variables) are statistically independent. While this assumption is often violated, it permits naive Bayes algorithms to work quickly and efficiently, which often yields benefits that outweigh the costs of the independence assumption. To implement a naive Bayes algorithm, the only definite requirement is that class labeled data instances are available for algorithm training and testing. If this condition is satisfied, implementation is executed through three steps:

1. Splitting labeled data into training and test data sets
2. Training the algorithm on the training data
3. Estimating the trained algorithm's error rate by testing it on the test data set

If a project step also includes classifying unlabeled data instances, a fourth step is required:

4. Using the trained algorithm to probabilistically assign a class to a set of unlabeled data instances.
Basic Tutorial: Using TACIT to implement a naive Bayes algorithm
TACIT's naive Bayes tool can be used to learn probabilistic associations between text instances and class labels. During the training process, the algorithm extracts features from training texts that are predictive of class association. The extraction of these features is both automatic and optimized, so it is not necessary to specify the number features to be extracted.
Data organization and specification
TACIT's naive Bayes tool requires that data organization satisfies two criteria. First, every text instance must be contained in its own file. Second, class labels must be represented by directories. That is, a directory must be created for each class label and all text instances associated with a given label must be located in the relevant directory.

Once the data is organized into single instance files contained in class directories, the class directories can be added to the naive Bayes implementation by clicking the "Add Folder" button, which is located in the naive Bayes tool's "Class Details" section. Using the "Add Folder" button, each class directory should be added to the implementation. Depending on whether the text instances need to be preprocessed, the "Preprocess" option, also contained in the "Class Details" section can be selected.

Notably, TACIT's naive Bayes tool does not require that data be split into training and test sets. When the algorithm is run, the data contained in the class directories is automatically split into training and test sets, following a 90% to 10% ratio, respectively.

In order to use a trained algorithm to classify unlabeled data, it is also necessary to organize the unlabeled data. Each instance must be contained in a separate file and all files must be stored in a single directory. The "classify data" option must then be selected and the directory that contains the unlabeled data must be specified as the "Classification Input Path." Both of these fields are located in the "Classification" area of the naive Bayes window in TACIT.
Specify k-value for cross validation
In order to estimate the accuracy of the classifier, TACIT's naive Bayes module uses k-fold validation. K-fold validation randomly splits a data set into k equal sized subsamples. One of these subsamples is then randomly selected as the test set and the algorithm is trained on the remaining k-1 samples. This process is then repeated k-times (the folds) so that each subsample is used once as the test set. The error rates obtained from each fold are then averaged to estimate the actual error rate. Any number up to the total sample size can be specified as the k-value; however, k=10 is one of the most commonly used values.

To specify the k value, enter the desired value in the "k value for cross validation" field, which is located in the "Input Parameters" field within the TACIT naive Bayes module window.
Output and implementation
If unlabeled data classification is desired, an output path must be specified. After the algorithm runs, a .csv file with the classification data will be generated. This file will contain a row for every unlabeled data instance and row will contain the filename of that row's instance, the assigned class, and other relevant information.

After specifying all requisite parameters, the algorithm can be run by clicking the green and white play button located in the top right corner of the naive Bayes window. Output information regarding the k-fold validation process as well as the average test accuracy, standard deviation, and standard error will be displayed in the console window.
Summary of steps
1. Store each text instance in its own text file
2. Create a directory for each class and store each associated data instance file in the appropriate directory
3. Select all relevant class directories within the naive Bayes window using the "Add Folder" button
4. If preprocessing is desired, select "Preprocess" and specify the required parameters
5. If unlabeled data classification is desired, store each unlabeled data instance in an individual file, place the files in a single directory, and specify this directory as the "Classifcation input path" using the "Browse" button
6. Specify the desired number of folds for k-fold validation
7. If unlabeled data classification is desired, specify the output path in which the classification .csv should be stored
