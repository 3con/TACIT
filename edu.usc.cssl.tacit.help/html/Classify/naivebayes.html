TACIT User Guide

TACIT Naive Bayes Classifier Tool
Overview
Naive Bayes methods are a family of supervised learning algorithms that are used to classify text files according to a user-determined set of class types. Naive Bayes classification is accomplished by applying Bayes theorem with the strong assumption that dependent features (i.e. dependent variables) are statistically independent. While this assumption is often violated, it permits naive Bayes algorithms to work quickly and efficiently, which often yields benefits that outweigh the costs of the independence assumption. To implement a naive Bayes algorithm, the only definite requirement is that class labeled data instances are available for algorithm training and testing. If this condition is satisfied, implementation is executed through three steps:

1. Splitting labeled data into training and test data sets
2. Training the algorithm on the training data
3. Estimating the trained algorithm's error rate by testing it on the test data set

If a project step also includes classifying unlabeled data instances, a fourth step is required:

4. Using the trained algorithm to probabilistically assign a class to a set of unlabeled data instances.
Basic Tutorial: Using the Naive Bayes Tool
The Naive Bayes tool can be used to learn probabilistic associations between text instances and class labels. During the training process, the algorithm extracts features from training texts that are predictive of class association. The extraction of these features is both automatic and optimized, so it is not necessary to specify the number features to be extracted.
Specifying Input Text for Analysis
All input data files must be saved as a corpus in Corpus Management or in .txt file format to be compatible with TACIT. See the Corpus Management Help Section if you need to convert files to compatible formats.

To specify which files you would like to analyze, select Add Corpus,Add Folder, or Add File(s) under the Input panel. All files and folders added to the input panel are automatically selected to be included in analysis by the check mark box to the left of the corpus/file/folder name. The number of files selected for analysis is indicated at the bottom of the input panel. To de-select an unwanted folder or file, uncheck the box next to its name. To remove a file completely from the program list, click on the file name to highlight it, and then click the "Remove" button. Note: Files within folders/corpora cannot be removed from the tool without removing the entire folder or corpus, but de-selecting files using the check boxes will remove them from analysis.

Add Corpus: The Add Corpus button will allow you to add a stored corpus from Corpus Management and all included sub-groups and fies for analysis. To expand the corpus and view the subgroups/files included, click on the arrow to the left of the corpus name in the input panel.
Add Folder: The Add Folder button will allow you to add a folder and all included subfolders and files for analysis. To expand the folder and view the subfolders/files included, click on the arrow to the left of the folder name in the input panel.
Add File(s): The Add Files button will allow you to add .txt files to be included for analysis. Multiple files within the same folder can be selected at the same time using standard multi-select functions.
Notably, the naive Bayes tool does not require that data be split into training and test sets. When the algorithm is run, the data contained in the class directories is automatically split into training and test sets, following a 90% to 10% ratio, respectively.

In order to use a trained algorithm to classify unlabeled data, it is also necessary to organize the unlabeled data. Each instance must be contained in a separate file and all files must be stored in a single directory. The classify data option must then be selected and the directory that contains the unlabeled data must be specified as the Classification Input Path. Both of these fields are located in the Classification area of the naive Bayes window in TACIT.
Specify k-value for cross validation
In order to estimate the accuracy of the classifier, thes naive Bayes module uses k-fold validation. K-fold validation randomly splits a data set into k equal sized subsamples. One of these subsamples is then randomly selected as the test set and the algorithm is trained on the remaining k-1 samples. This process is then repeated k-times (the folds) so that each subsample is used once as the test set. The error rates obtained from each fold are then averaged to estimate the actual error rate. Any number up to the total sample size can be specified as the k-value; however, k=10 is one of the most commonly used values.

To specify the k value, enter the desired value in the k value for cross validation field, which is located in the Input Parameters field within the TACIT naive Bayes module window.
Output and implementation
To specify an output folder where crawled files will be saved, click on the Browse button to the right of the Output Location bar and select a folder. If you create a new folder within this menu and change its name from "New Folder", click on any other folder and then click back on your newly created & renamed folder to select it. After specifying all parameters, click the green and white play button located in the top right corner of the window to run the program. Output information will display in the console panel at the bottom of the tool.

If unlabeled data classification is desired, an output path must be specified. After the algorithm runs, a .csv file with the classification data will be generated. This file will contain a row for every unlabeled data instance and row will contain the filename of that row's instance, the assigned class, and other relevant information.

After specifying all parameters, click the green and white play button located in the top right corner of the window to run the program. Output information regarding the k-fold validation process as well as the average test accuracy, standard deviation, and standard error will be displayed in the console panel at the bottom of the tool.
Summary of steps
1. Store each text instance in its own text file
2. Create a directory for each class and store the data instance files created in step one in the appropriate directory
3. Select all relevant class directories within the naive Bayes window using the Add Folder button
4. If preprocessing is desired, select Preprocess and specify the required parameters
5. If unlabeled data classification is desired, store each unlabeled data instance in an individual file, place the files in a single directory, and specify this directory as the Classifcation input path using the Browse button
6. Specify the desired number of folds for k-fold validation
7. If unlabeled data classification is desired, specify the output path in which the classification .csv should be stored
Understanding Naive Bayes Output
Training and test output
When TACIT's naive Bayes module is implemented performance statistics for each k-fold validation will be output in the Console window. The average test accuracy (averaged across k-folds), as well as its standard deviation and standard error, will also be output in the Console window. These statistics should be used to evaluate the performance of the algorithm on the data provided.

Each time the module is implemented a log file will be created in the directory that contains the class directories. This file will be titled "Naive_Baye's-run-report-Date-Time.txt" and it will contain the TACIT version number and the date and time. Naive-Bayes's-run-report-07-29-15-10-44-46.txt
Classification output
If unlabeled data is classified during the naive Bayes implementation a classification file will be created in the directory that contains the class directories. The title of this file will be the names of the classes followed by the date and time (e.g. "class1-class2-Date-Time.csv"). Classification files created by TACIT's naive Bayes module contain N rows for each of the N instances that were classified and four columns. The columns are: "File," which contains the file name of each instance; "Predicted Class," which contains the class predicted for instance N; "Predicted Class Probability," which contains the estimated probability that instance N belongs to the predicted class; and "Other Class Probabilities," which contains the estimated probabilities of instance N belonging to the classes that were not predicted.
