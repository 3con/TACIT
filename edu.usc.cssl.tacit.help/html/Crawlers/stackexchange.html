TACIT User Guide

TACIT StackExchange Crawler
Overview
The StackExchange Crawler provides full access to all data available on stackexchange.com, a question and answer website covering topics in various fields. This crawler provides various filtering options that can be used to search for specific Q/A domains and tags. This data is then saved in a format that is accessible and analyzable by the TACIT tool as well as other language analysis algorithms.

Basic Tutorial: Collecting data using the StackExchange Crawler
Selecting Input Parameters
The Select Domains input box lets you specify specific Q/A web site domains to which you would like to limit your search. You must specify a domain in order to crawl.

The Search by Tag input box allows you to specify tags (or keyword/keywords) you would like to base your search on. The crawler will return posts that have been marked with these specified tags. Note: StackExchange does not allow keyword searching of the question, answer, or comment text directly at this time.

Stored Attributes Filters
You can further tailor and filter your crawling results using the following options:
Collect Answer/Comment Text: These check boxes allow you to specify whether you would like to collect the answers and comments associated with the questions. If you select these boxes, the crawler will default to collecting 5 answers and 5 comments per question.
User Details for Questions/Answers/Comments collect information about the users whose text you are collecting including user id, username, user reputation, and whether or not they are registered.
Is Question Answered: Selecting this box will limit results to only questions with answers.
Limit Results
The Specify Date Range allows you to specify the time frame that you would like to receive data for based on calendar date.

Changing the numbers in the Limit Questions Per Request, Limit Answers Per Question, and Limit Comments Per Question text boxes will allow you to change these default numbers. Note that one question page returns 35 questions, and the default is 2 pages per request, 5 answers per question, and 5 comments per question.

Crawl Order will let you specify whether you would like the results to be collected started with the most recent questions that match your request or if you would prefer to have the crawl return a random subset of all questions that fit your parameters regardless of post date.
Specifying Output Details
By default, all collected text in TACIT is saved as a corpus in Corpus Management. Specify the name you would like to assign this corpus in the Corpus Name box under Output Details .

Understanding StackExchange Crawler Output
Crawl data is saved in Corpus Management under the corpus name you provided. Data can be output into formats compatible with other programs using Corpus Management.
