TACIT User Guide

Frontiers Journal Crawler
Overview
The Frontiers Journal Crawler tool collects text from the Frontiers in Psychology website, a site that provides an Open Science platform and publishes many widely known open-access journals. The crawler allows you to search for data pertaining to specific fields including Science, Health, Engineering, and Humanities and Social Sciences. Then based on these fields, you select more specific sub-domains. The tool also allows you to further filter your results by limiting the number of pages being collected. The data is then saved in a format that is accessible and analyzable by the TACIT tool as well as other language analysis algorithms.

Basic Tutorial: Collecting data using the Frontiers Journal Crawler
Selecting Crawler Filters
The Select Field option allows you to specify which field you would like the crawler to collect data from within the different journals on the website. The options you are given when later selecting the sub-domain will change based on the initial field you select here. Your options are: Science, Health, Engineering, and Humanities and Social Sciences.

The Select Sub-Domains option allows you to further narrow the type of data collected from the various journals. You must press the Add button to pull up your choices of sub-domains and apply them to the search. In order for the crawler to run, you must select at least one sub-domain.

You can limit the number of pages being collected in the Filter Results section. Once you select the check box next to the option, you can enter the number of records per topic you want.

Specifying Output Details
By default, all collected text in TACIT is saved as a corpus in Corpus Management. Specify the name you would like to assign this corpus in the Corpus Name box under Output Details .
Understanding Frontiers Crawler Output
The Console panel at the bottom of the tool will provide information about the Supreme Court term/issue selected, the website url being crawled, and the names of the files being created while the crawler is running. Crawl data is saved in Corpus Management under the corpus name you provided. Data can be output into formats compatible with other programs using Corpus Management.
