TACIT User Guide

Hansard United Kingdom Parliament Crawler
Overview
The Hansard United Kingdom Parliament Crawler collects text from The Hansard Parliament Website, which contains the official report of all parliamentary debates. The crawler writes that data into text files that are readable by automated text analysis programs.

Basic Tutorial: Collecting data using the Hansard United Kingdom Parliament Crawler
Selecting Parliament Documents
The Hansard crawler can collect data by either a user specified Keyword Search or by a search of House Members

Keyword Search: The keyword search allows you to crawl the website for reports containing specified keywords. You may check the box to limit by a date range and then specify the from and/ or to dates in the following boxes in the format MM/DD/YYYY. You may also choose to limit the number of results crawled to a desired number.
House Member Search:The house member search does not allow you to enter a keyword, but instead use the following House Member Search segment. Here, you can enter the name of one or more house members, as well as the number of articles you desire from each house member. You may also opt to enter the name of a different Congress Member in addition or instead of the house member search.
Specifying Output Details
By default, all collected text in TACIT is saved as a corpus in Corpus Management. Specify the name you would like to assign this corpus in the Corpus Name box under Output Details .
Understanding Presidential Paper Crawler Output
The Console panel at the bottom of the tool will provide information about the Supreme Court term/issue selected, the website url being crawled, and the names of the files being created while the crawler is running. Crawl data is saved in Corpus Management under the corpus name you provided. Data can be output into formats compatible with other programs using Corpus Management.
